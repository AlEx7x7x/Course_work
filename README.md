# Course_work
**ІНСТРУКЦІЯ З РОЗГОРТАННЯ ТА ВИКОРИСТАННЯ**

Цей документ описує процедуру встановлення, налаштування та використання веб-застосунку SwiftRoute.

**1. Системні вимоги**
Для коректного функціонування програмного комплексу на цільовому комп’ютері має бути встановлене наступне програмне забезпечення:

Операційна система: Windows 10/11, macOS або Linux.

Платформа Node.js: Рекомендована версія LTS (16.x або вище). Перевірити версію можна командою node -v у терміналі.

СУБД MySQL Server: Версії 8.0. Рекомендовано використовувати MySQL Workbench для імпорту даних та адміністрування.

Веб-браузер: Google Chrome, Mozilla Firefox або Microsoft Edge (останні версії).

**2. Підготовка бази даних MySQL**
Перед запуском серверної частини Next.js необхідно підготувати схему бази даних та імпортувати GTFS-дані, оскільки вони є основним джерелом інформації.

Створення схеми: Запустіть MySQL Workbench. Створіть нову схему (Schema) з назвою, наприклад, gtfs_db.

Імпорт даних:

Якщо ви використовуєте database_dump.sql, виконайте імпорт через Data Import/Restore у Workbench, вказавши цільову схему gtfs_db.

Якщо ви імпортуєте GTFS-файли вручну (.txt), використовуйте інструмент імпорту CSV для завантаження таблиць (routes, stops, shapes тощо) у схему gtfs_db.

Верифікація: Переконайтеся, що всі таблиці (близько 6-8) імпортовані та містять дані.

**3. Налаштування та запуск застосунку (Next.js)**
Проєкт SwiftRoute є єдиним монолітним застосунком на базі Next.js, який поєднує клієнтську та серверну логіку (SSR).

Встановлення залежностей: Відкрийте термінал (командний рядок) у кореневій директорії проєкту та виконайте команду:


```
npm install
```

Налаштування підключення до БД:

Знайдіть файл конфігурації бази даних (зазвичай lib/db.js або lib/db.mjs).

Вкажіть у ньому параметри підключення, створені на Кроці 2.1:

JavaScript
```
// Фрагмент коду (приблизно в lib/db.js)
const connection = mysql.createConnection({
    host: 'localhost',
    user: 'root',
    password: 'ваш_пароль_mysql', // <-- ВКАЗАТИ ВАШ ПАРОЛЬ!
    database: 'gtfs_db' 
});
```
Запуск застосунку: Запустіть застосунок у режимі розробки:

Bash
```
npm run dev
```
Доступ до інтерфейсу: Після успішного запуску (який може зайняти час, оскільки Next.js виконує компіляцію та перше підключення до БД), відкрийте браузер та перейдіть за адресою:
```
http://localhost:3000
```
**4. Посібник користувача та тестування функціоналу**
**4.1. Основна навігація та фільтрація**
Вибір типу транспорту: У верхній частині бічної панелі використовуйте випадаючий список, щоб фільтрувати маршрути за типом (Трамваї (0), Тролейбуси (400), Автобуси (3)).

Пошук маршруту: Введіть номер або частину назви маршруту у поле пошуку. Список оновлюється миттєво (завдяки useMemo).

Активація маршруту: Натисніть на картку маршруту в списку. Це:

Встановлює стан activeRouteId.

Ініціює відображення його траєкторії (Polyline) на карті.

Ініціює відображення відповідних зупинок.

**4.2. Візуалізація та прогнози**
Перегляд зупинок та розкладу: Натисніть на маркер зупинки на карті. У спливаючому вікні відобразиться список найближчих прибуть для цієї зупинки, використовуючи комбіновані дані GTFS Realtime (якщо доступні) та статичний розклад.

Симуляція руху: Активуйте кнопку "Увімкнути Симуляцію". На карті з'являться анімовані маркери, що імітують рух транспорту згідно з розкладом, демонструючи функціонал інтерполяції координат.
